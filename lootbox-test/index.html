Обираєте, що «поверне бекенд» при кліку на активну картку, і затримку відповіді. Пісочниця ловить cardClick і через задану затримку надсилає setCardState — так само, як робитиме ваш реальний бекенд. Віджет крутить «зарядку» кулі, поки чекає, а потім спалахом відкриває результат.

Тільки для тесту

Ці команди не використовуються в продакшен-інтеграції. Вони потрібні лише для ручного тестування віджета в цій пісочниці.

Кнопка Надіслати playOpen для кожної картки — у відповідному рядку вище. Запускає draft-анімацію без реального кліку (тільки для тесту).

Кроки інтеграції (шпаргалка)

Стислий флоу. Повний контракт — у INTEGRATION.md. Popup, blur і UI поза каруселлю реалізує FE на prod-сторінці.

  1. Вставити iframe з даними карток у query-параметрах або короткий URL + setContent після ready (для великих/динамічних наборів).
  2. Слухати ready; на resize — оновити висоту iframe.
  3. На cardClick (available) — одразу робити запит до API. Віджет крутить «зарядку» кулі, поки ви чекаєте відповідь.
  4. Коли API відповів — надіслати setCardState з результатом (prize/prediction). Віджет спалахом відкриває результат (конфетті лише для prize).
  5. На animationComplete (після спалаху) — показати перший popup результату.
  6. Повторний клік по сьогоднішньому prize/prediction — знову popup (без анімації).

⚠️ Коли рендерити popup: cardClick приходить у двох випадках — розрізняйте за data.state. На state:'available' (перший клік) popup НЕ показуйте — лише стартуйте запит до API (результату ще немає). Перший popup — тільки на animationComplete. На повторний cardClick зі state:'prize'/'prediction' — можна показати popup знову. Ніколи не вішайте popup на «будь-який cardClick».
